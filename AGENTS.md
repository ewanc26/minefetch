# AGENTS.md

Guidance for agents working on Minefetch, a Kotlin PaperMC plugin that displays host system information generated by external Fastfetch watcher scripts.

## Boundaries

- `src/main/kotlin/` contains plugin lifecycle, command handling, safe sysinfo reading, and update checking.
- `src/main/resources/plugin.yml` defines commands and permissions.
- `scripts/` contains macOS/Linux/Windows installation and watcher jobs plus Fastfetch configuration. The plugin reads their output; it must not shell out on every command.

## Invariants

- Treat the sysinfo file as untrusted, concurrently replaced input. Bound its size, decode safely, and handle missing/partial files.
- Do not expose secrets, usernames, internal IPs, or arbitrary host files to players.
- Keep file I/O and update checks off the server thread; send Paper messages on the main thread.
- Install scripts must be idempotent and avoid overwriting unrelated launchd/tasks/files.
- Maintain equivalent data keys across shell and PowerShell watchers.

## Validation

Run `./gradlew build`. Run `shellcheck`/`bash -n` for shell changes and parse-check PowerShell where available. Test plugin enable, permissions, missing/stale/malformed/oversized sysinfo, atomic watcher updates, reload, and update-check failure on a test Paper server. Never commit generated host data or build output.
